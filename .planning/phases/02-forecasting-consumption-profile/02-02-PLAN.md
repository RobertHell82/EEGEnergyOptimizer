class PVForecast:

class ForecastProvider:

    def get_forecast(self) -> PVForecast: ...
    @staticmethod

class SolcastProvider(ForecastProvider):
    def __init__(self, hass, remaining_entity: str, tomorrow_entity: str): ...

class ForecastSolarProvider(ForecastProvider):
    def __init__(self, hass, remaining_entity: str, tomorrow_entity: str): ...

       - Parameterized by `day_offset` (0=today remaining, 1=tomorrow, 2-6=day+2..+6)

       - `_attr_native_unit_of_measurement = UnitOfEnergy.KILO_WATT_HOUR`
       - `_attr_device_class = SensorDeviceClass.ENERGY`
       - `_attr_icon = "mdi:lightning-bolt"`
       - Do NOT set `_attr_state_class` (these are forecasts, not measurements)
       - `async_update()`: For day_offset=0, calculate_period(now, end_of_today). For 1-6, calculate_period(start_of_day, end_of_day).
       - 7 instances total (day_offset 0 through 6)

       - `_attr_name = "Prognose bis Sonnenaufgang"`
       - `async_update()`: Read `sun.sun` entity attributes `next_rising`, use coordinator.calculate_period(now, next_sunrise)
       - If sun entity unavailable or next_rising is None, set native_value=None

    4. `BatteryMissingEnergySensor` (fast, per D-14):
       - `_attr_name = "Batterie fehlende Energie"`
       - `_attr_native_unit_of_measurement = UnitOfEnergy.KILO_WATT_HOUR`
       - `_attr_device_class = SensorDeviceClass.ENERGY`
       - `_attr_icon = "mdi:battery-charging-outline"`
       - `_attr_suggested_display_precision = 2`

    5. `PVForecastTodaySensor` (fast, per D-15):
       - `_attr_name = "PV Prognose heute"`
       - Reads `provider.get_forecast().remaining_today_kwh`

    6. `PVForecastTomorrowSensor` (fast, per D-15):
       - `_attr_name = "PV Prognose morgen"`
       - Reads `provider.get_forecast().tomorrow_kwh`
