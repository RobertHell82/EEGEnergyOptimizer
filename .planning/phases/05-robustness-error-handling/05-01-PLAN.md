Three changes in async_setup_entry():

**Change 1 — Add logging import at top of file (after existing imports):**
```python
import logging

_LOGGER = logging.getLogger(__name__)
```

**Change 2 — Wrap create_inverter call (around line 110) in try/except:**

Replace:
```python
    inverter = create_inverter(config.get("inverter_type", ""), hass, config)
    hass.data[DOMAIN][entry.entry_id]["inverter"] = inverter
```

With:
```python
    try:
        inverter = create_inverter(config.get("inverter_type", ""), hass, config)
    except ValueError as err:
        _LOGGER.error("Failed to create inverter: %s", err)
        from homeassistant.components.persistent_notification import async_create
        async_create(
            hass,
            f"EEG Optimizer: Wechselrichter konnte nicht erstellt werden — {err}",
            title="EEG Optimizer Fehler",
            notification_id="eeg_inverter_error",
        )
        return False
    hass.data[DOMAIN][entry.entry_id]["inverter"] = inverter
```

Returning False from async_setup_entry tells HA the integration failed to load, which surfaces the error in the UI.

**Change 3 — Add else branch to coordinator/provider check (around line 121):**

After the existing `if coordinator and provider:` block (which ends around line 139), add an else branch:

```python
    else:
        missing = []
        if not coordinator:
            missing.append("Verbrauchsprofil (coordinator)")
        if not provider:
            missing.append("PV-Prognose (provider)")
        _LOGGER.error(
            "EEG Optimizer: Optimizer konnte nicht gestartet werden — "
            "fehlende Komponenten: %s",
            ", ".join(missing),
        )
        from homeassistant.components.persistent_notification import async_create
        async_create(
            hass,
            f"EEG Optimizer konnte nicht vollstaendig starten. "
            f"Fehlende Komponenten: {', '.join(missing)}. "
            f"Bitte Setup-Wizard erneut durchlaufen.",
            title="EEG Optimizer Warnung",
            notification_id="eeg_init_warning",
        )
```

Note: The integration still loads (returns True) so the panel remains accessible for troubleshooting. But the user gets a persistent notification AND a log error explaining what is missing.
