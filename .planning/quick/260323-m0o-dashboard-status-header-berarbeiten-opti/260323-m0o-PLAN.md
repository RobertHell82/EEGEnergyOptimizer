**Morning delay status computation** (add a new method `_morning_delay_status(self, snap: Snapshot) -> dict`):
1. If `not self._enable_morning_delay`: status="deaktiviert"
2. Compute `in_window` (sunrise-1h <= now <= morning_end), `threshold = bedarf * (1+buffer)`, `pv_today = snap.pv_remaining_today_kwh`
3. If `in_window` and `pv_today > threshold`: status="aktiv", reason="Ladung blockiert bis {morning_end}"
4. If `in_window` and `pv_today <= threshold`: status="nicht_aktiv", reason="PV reicht nicht fuer Bedarf + Puffer"
5. If not `in_window`:
   - Need to check if tomorrow's PV would exceed tomorrow's demand + buffer for "morgen_erwartet" vs "morgen_nicht_erwartet"
   - Use `snap.pv_tomorrow_kwh` and compute tomorrow demand estimate from `snap.consumption_tomorrow_kwh + missing_battery_estimate`
   - Tomorrow threshold = tomorrow_demand * (1 + buffer%)
   - If pv_tomorrow > tomorrow_threshold: status="morgen_erwartet", reason="Morgen ab ~{sunrise_time}"
   - Else: status="morgen_nicht_erwartet", reason="PV Prognose zu gering"
   - For sunrise display: use snap.sunrise formatted as HH:MM (this is next_rising which IS tomorrow's sunrise when checked in afternoon/evening; when between midnight and sunrise it is today's sunrise — both are fine for display)

**Discharge status computation** (refactor within `_evaluate()`, after calling `_should_discharge()`):

   - Filter reasons: separate time-reason ("Startzeit...nicht erreicht") from condition-reasons
   - If there are ONLY time-reasons (no SOC/PV failures): status="geplant"
   - If there are condition-reasons: status="nicht_geplant", populate discharge_reasons with the condition failures only (not the time one)
4. Populate all discharge_* fields from snap values and computed min_soc
