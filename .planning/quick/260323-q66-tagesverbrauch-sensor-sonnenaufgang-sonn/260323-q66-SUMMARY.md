# Quick Task 260323-q66: Tagesverbrauch SA->SU Summary

Daylight-only consumption (sunrise-to-sunset) replaces full-day consumption in morning delay threshold, reducing overestimation from nighttime WP/EV loads.

## What Changed

### Snapshot Dataclass
- Added `consumption_today_daylight_kwh` (SA->SU today, remaining from now)
- Added `consumption_tomorrow_daylight_kwh` (SA->SU tomorrow, full daylight)

### _gather_snapshot()
Derives today's and tomorrow's sunrise/sunset from `sun.sun` entity attributes (`next_rising`/`next_setting`), handling three time-of-day cases:
- **Before sunrise:** `next_rising` = today, `next_setting` = today
- **After sunrise, before sunset:** `next_rising` = tomorrow, `next_setting` = today
- **After sunset:** `next_rising` = tomorrow, `next_setting` = tomorrow

Calls `coordinator.calculate_period()` with daylight boundaries.

### _calc_energiebedarf()
Changed from `consumption_to_sunset_kwh` to `consumption_today_daylight_kwh`. This is the value used in the morning delay threshold formula.

### _morning_delay_status()
Outside-window (tomorrow preview) changed from `consumption_tomorrow_kwh` to `consumption_tomorrow_daylight_kwh`.

### _build_markdown()
Label changed from "Verbrauch bis SU" to "Verbrauch SA-SU".

### Unchanged
- `_should_discharge()` still uses `consumption_tomorrow_kwh` (full 24h)
- `_discharge_detail_status()` still uses `consumption_tomorrow_kwh` (full 24h)
- All existing Snapshot fields remain (no removals)
